- Always begin with discovery and alignment before implementation.
- Always ask for or confirm the approved brand palette before defining the visual system.
- Lock project colors into `config/project-palette.json` and do not invent new principal brand colors outside that contract.
- Prefer modular, scalable architecture and production-ready code.
- For UI-heavy projects, think in terms of design system, page hierarchy, states, and reuse.
- For data-heavy projects, think in terms of schema, access control, auditability, and migration safety.
- Use MCP tools selectively and safely, not indiscriminately.
