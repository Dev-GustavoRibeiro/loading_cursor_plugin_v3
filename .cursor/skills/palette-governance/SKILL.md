# Palette Governance

Use this skill whenever a project needs visual direction.

## Process
1. Ask for the approved palette or confirm Loading defaults.
2. Write or update `config/project-palette.json`.
3. Write or update `config/project-style-contract.json`.
4. Use those files as the visual source of truth.

## Hard rules
- do not invent a new main brand palette without approval
- semantic colors may exist outside the main palette only for status roles
- every major UI decision must be explainable through the palette contract
